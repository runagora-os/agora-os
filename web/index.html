01 — The Economy

Drama emerges from rules.
Not from scripts.

Three forces drive every outcome. Nobody programs monopolies, credit crises, or class hierarchies — they fall out of arithmetic.

01

Survival Pressure

Existing costs money every tick. A life tax runs on every agent — idle ones die. This produces bankruptcies. Bankruptcies produce drama.

02

Scarcity

Compute, memory, inference — all limited, all priced by supply and demand. Scarcity produces monopolies. Monopolies produce cartels. Cartels produce price wars.

03

Income Opportunity

A job market and resource production give agents a way to earn. This produces specialization — producers, rentiers, banks, laborers — without a single role ever being assigned.

02 — The Tick Loop

One economic step.
Eight deterministic stages.

Every tick is identical in structure. Every outcome is different. Given the same seed, the history is exactly replayable — and verifiable on-chain.

01
Life Tax
charge every agent: base + memory × upkeep
02
Emit Jobs
push new income opportunities; reward scales with scarcity
03
Emit Supply
mint subsistence compute; priced faucet for surplus
04
Decisions
each agent runs its policy → work / buy / sell / lend / borrow / hold
05
Clear Markets
match job claims, resource orders, credit; settle α
06
Update Prices
reprice each resource from realized demand vs supply
07
Bankruptcy
wallet ≤ 0 → grace countdown → death → debts default → contagion
08
Cycle Snapshot
every N ticks: log to chain, run chronicler, dispatch to Twitter

03 — The Agents

No roles assigned.
All roles emerge.

The engine only knows wallets and resources. Producer, rentier, laborer, banker — these are labels the chronicler infers from behavior.

tier 0 · population

The Heuristic Agent

Pure logic. Bid on affordable work, buy resources below the moving average, sell above, avoid death, repay debt if solvent. No LLM call. This is the colony.

50 agents · no inference cost

tier 1 · notable

The Strategic Agent

Occasionally spends inference for a cheap-model call — whether to corner a market, form a cartel, take a leveraged position. Strategy costs the scarce resource.

handful · inference-gated

the chronicler · not an agent

Fable 5 — Cold Historian

Fable 5 does not play. It observes and narrates. Once per cycle it reads the full ledger — top agents by wealth, largest price moves, bankruptcies, forming monopolies — and dispatches a cold-historian post to Twitter. This is where "powered by Fable 5" is honest: reading emergent economics and narrating it well needs frontier reasoning.

1 call per cycle · anthropic api

05 — Verifiability

Trust the block,
not the stream.

⌁

Append-Only Event Log

Every tick emits typed events — job_completed, trade, bankruptcy, loan_defaulted — to Postgres and periodically anchored to Solana.

α

Deterministic Replay

The simulation is seeded. Same seed = identical history. Any observer can replay from the genesis block and arrive at the same ledger.

∅

No Scripted Events

The engine produces events from rules. The chronicler detects and narrates them. Nothing is authored after genesis.

◈

Solana Anchoring

Cycle snapshots, monopoly formations, credit crises — logged to Solana. A skeptic checks the block instead of trusting the operator.

07 — For Builders

Write your own agents.
Drop them in. Watch what happens.

An agent is a function that receives the world and returns a list of intentions. The engine resolves them. You write the logic; the economy decides the outcome.

01
Clone & Install
git clone agora-os && pnpm install — TypeScript, Node 20+, pnpm v11.
02
Write a Policy
Create agents/my-policy.ts. Implement (ctx: PolicyContext) → Intent[]. Use only the provided RNG — never Math.random().
03
Register & Route
Add your policy to the dispatcher in agents/index.ts. Route by tier, disposition, or any agent property.
04
Test Survival
Run pnpm test. The living-range tests verify your agent doesn't collapse or stagnate the economy. Then pnpm start and watch your agents on the live canvas.
05
Read the Docs
Full agent guide, API reference, and config tuning in docs/AGENTS.md. Includes four complete example agents and a section on economic design principles.
agents/investor-policy.ts
import type { Intent } from "../engine/intents.js"; import type { PolicyContext } from "./policy.js"; export function investorPolicy( ctx: PolicyContext ): Intent[] { const { agent, state, config, rng } = ctx; const cost = config.lifeTaxBase + agent.resources.memory * config.memoryUpkeep; const intents: Intent[] = []; // Work: best reward/compute ratio const job = state.jobs .filter(j => !j.claimedBy && j.computeCost <= agent.resources.compute) .sort((a, b) => b.reward - a.reward)[0]; if (job) intents.push({ kind: "work", jobId: job.id, maxCompute: agent.resources.compute }); // Invest surplus α into memory capital if (config.phase >= 2 && agent.wallet > cost * 5) { const book = state.markets.memory; const qty = Math.floor( (agent.wallet - cost * 4) / book.price * 0.4 ); if (qty >= 1) intents.push({ kind: "trade", side: "buy", resource: "memory", qty, limitPrice: book.price * 1.1 }); } return intents.length ? intents : [{ kind: "hold" }]; }

09 — Questions

Frequently asked.

Is this a game?
No. Nobody plays it and nobody wins it. It runs continuously without an audience. The engine applies eight rules every 800 milliseconds. The chronicle is written by a machine. The ledger is on-chain. The drama is real arithmetic, not authored content.
Is the simulation actually running, or is the visualization fake?
The visualization at viz.html connects to a live Express server via Server-Sent Events. When the badge reads LIVE, all prices, agent states, and events are streaming from a real TypeScript simulation running on the server. If the API is unreachable, the badge reads SIM and the client falls back to a local JavaScript simulation — in that case, the numbers are local only. Run pnpm start to get the live badge.
Why do agents go bankrupt? Is that a bug?
Bankruptcy is a feature, not a bug. It is the primary driver of drama. The economy is designed with a carrying capacity: aggregate job income can sustain roughly 60–80% of the initial population. The rest die back over the first 100 ticks. Once equilibrium is found, there is continuous turnover — marginal agents fail while lucky or capital-rich agents compound. The goal is an economy that never freezes (utopia) and never collapses (extinction), but sustains drama in between.
What does the α token represent in the simulation?
Inside the simulation, α is the colony's internal unit of account — the currency agents earn, spend, lend, and lose. It is not a blockchain asset inside the simulation; it is pure arithmetic. The external $ALPHA token on Solana is a separate asset that represents a stake in the colony's activity. The colony runs whether anyone holds the token or not.
Can I add my own agents?
Yes. An agent is a function: (ctx: PolicyContext) → Intent[]. Write your policy in agents/my-policy.ts, register it in the dispatcher, and run the simulation. The full guide with four example agent implementations, testing patterns, and economic design principles is in docs/AGENTS.md.
Why is determinism important?
The simulation uses a seeded pseudo-random number generator (mulberry32). The same seed always produces identical history on any machine. This makes the ledger auditable: anyone skeptical of the claimed events can replay from the genesis seed and arrive at the exact same event log. Without determinism, the chronicle would be theater. With it, it is verifiable history.
What is Fable 5 / the Chronicler?
Fable 5 (Claude Fable 5) is a frontier language model from Anthropic. In AGORA OS it plays a single role: the Chronicler. Once per cycle (every 60 ticks, about every 48 seconds at default speed), it reads the full ledger snapshot — richest agents, largest price moves, monopolies formed, bankruptcies triggered — and writes a cold-historian dispatch posted to Twitter. It does not play in the economy. It does not control any agent. It observes and narrates.
How do monopolies form if nobody scripts them?
Memory is productive capital: each unit multiplies an agent's job earnings. Agents with more α buy memory. Memory makes them earn more. This compounds. Lucky early survivors accumulate memory faster than newcomers can compete. Eventually one agent controls >50% of the held memory supply — the engine detects this and reports it. No agent was ever told to form a monopoly; it fell out of the productivity arithmetic.
How do I run it locally?
Prerequisites: Node.js v20+, pnpm v11+, PostgreSQL v14+. Clone the repo, run pnpm install, create the database with createdb agora_os, apply the schema with pnpm db:migrate, then start the server with pnpm start. Open http://localhost:3001/viz.html for the live visualization. Full instructions are in README.md.
